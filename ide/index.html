Development tools (installation and usage)

Content of the tutorial

The aim of this tutorial is to describe how to get started with the development tools we use at Everit. Please follow each step carefully! Do not worry if you do not know any of the technologies, they will be introduced in later chapters. When reading this section, please focus on the IDE, its settings and its usage only.

Part 1: Installing Java

Installing Java on Linux

On Linux, the newest OpenJDK package has to be downloaded and installed from the repository of the given Linux distribution.

Installing Java on Windows

  1. Download the latest Java SDK from http://www.oracle.com. In case your system is 64bit compatible, it is recommended to download the x64 version.

  2. Install Java by running the exe file you downloaded. Normally, you have to press "next", "next" and "finish". The default installation path of java is "c:/Program Files/java".

  3. Add the environment variable "JAVA_HOME" with the value of the location of your Java instance. Do not put slash after the last character of the file path!

  4. Extend the PATH environment variable with "%JAVA_HOME%\bin;". You should put this before all the other values of PATH. The reason is that a java.exe is created in the directory c:\Windows\System32 after installation. In case a JRE is installed with a newer version, this java.exe will be replaced. If the c:\Windows\System32 folder is before the %JAVA_HOME%\bin value in the PATH environment variable, it may happen that the first java.exe in the PATH is different from the one located in JAVA_HOME. This would cause a lot of inconveniences.

Part 2: Installing Maven

Maven is the build tool Everit projects are compiled with. It can be installed as described below:

  1. Download the latest stable build of Maven from http://maven.apache.org

  2. Unzip maven into a directory of your choice

  3. In windows:

    • Add M2_HOME to the environment variables where the value is the path of the folder of your Maven installation, without a slash after the last character of the file path. Do not worry if you use Maven3, the name of the environment variable is still M2_HOME.

    • Append the value of the PATH environment variable with "%M2_HOME%\bin"

    In Linux:

    • Create a symbolic link that points to the mvn script file at /usr/local/bin

Part 3: Installing Eclipse

Download the latest stable version of Eclipse. You will need the "Eclipse IDE for Java EE developers" version. It is a platform dependent ZIP file that you can uncompress into any folder. You can start Eclipse with the binary file located in the root directory of Eclipse (eclipse.exe in Windows or eclipse in Linux).

The JEE version of Eclipse should already contain the Git and the M2E Eclipse plugins, which are necessary to continue this tutorial.

Customizing Java editor

To set up the following preferences is strongly recommended, but it is not required only if one wishes to contribute to the codes written by Everit.

  1. Click the Window/Preferences menu item in the Eclipse menu bar.

    • Type "Text Editors" in the "type filter text" field and click "Text Editors".

    • Tick "Show print margin" in the "Text Editors" menu and set the value of "Print margin column" to 100.

    • Tick "Show line numbers".

    • Tick "Show whitespace "

  2. Click "Apply" and then "OK".

Customizing Project Explorer

By default, Project Explorer does not show generated folders and resources starting with a dot. As it is often necessary to check the content of these files (especially the content of the generated MANIFEST.MF file), the settings of the Project Explorer should be modified.

  1. Select the Customize view... menu item from the drop down menu of Project Explorer

  2. Remove the tick from .*resources and Java output folders

  3. Push the OK button

Configuring Java formatter

  1. Use "Save link as.." to download the following file containing the required formatter settings:

    everit-eclipseformatter.xml

  2. Click the Window/Preferences menu item in the Eclipse menu bar.

  3. Type "formatter" in the "type filter text" field and select "Java/Code Style/Formatter" menu item.

  4. Click the "Import" button and select the previously downloaded file.

  5. Click "OK" to make the above file the default formatter.

Configuring "Clean up" and "Save Actions"

  1. Use "Save link as.." to download the following file containing the required clean up and save actions settings:

    everit-clean-up-and-save-actions.xml

  2. Click the Window/Preferences menu item in the Eclipse menu bar.

  3. Type "clean up" in the "type filter text" field and select the "Java/Code Style/Clean Up" menu item.

  4. Click the "Import" button and select the previously downloaded file.

  5. Click "OK" to make the above file the default code cleaner.

  6. To clean up a code, click "Source/Clean Up..." when in a java file or set up the automatic "Save action" in the same way:

Configuring the XML/HTML formatter

  1. Click the Window/Preferences menu item in the Eclipse menu bar.

  2. Type "XML Editor" in the "type filter text" field and click "Editor".

  3. Set "Line width" to 100.

  4. Uncheck "Format comments".

  5. Tick "Indent using spaces" and set its value to 2.

  6. Click "Apply" and then "OK".

  7. Do the same for HTML editors

Installing m2e connectors

To keep the target/classes directory up-to-date and having properly configured projects after a project import and auto-configure code quality plugins of Eclipse, a couple of m2e connectors have to be installed.

  1. Select Window/Preferences in your Eclipse IDE

  2. In the preferences window, go to Maven/Discovery and click on the "Open Catalog" button

  3. In the popup window, select following extensions:

    • m2e-checkstyle

    • m2e-findbugs

    • m2e-mycila-license

    • m2e-pmd

    • Tycho Configurator

  4. On the following screen, click Next

  5. Click Next again

  6. Accept the licenses and Finish the installation

  7. Restart Eclipse or continue with the next steps

Adding Everit extensions for Checkstyle

Getting Everit extensions for Checkstyle

  1. Download the newest version of Everit Coding Conventions JAR from maven central

  2. Copy the jar file into the dropins folder of Eclipse

  3. Restart Eclipse

Installing JaCoCo Eclipse plugin

During our development it is very useful to see what is tested. This tutorial uses JaCoCo, a great code coverage tool that has an Eclipse plugin as well.

  1. Click the Help/Eclipse Marketplace... menu item in the Eclipse menu bar and select the "Search" menu.

  2. Type "jacoco" in the "Find:" field and click "Install" as shown below.

  3. Select all components, click "Confirm", accept the license agreement and finish installation.

  4. Restart Eclipse to complete installation.
