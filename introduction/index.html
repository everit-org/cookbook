Everit development guide

Introduction

This documentation is the curriculum of Everit Ltd. When a new colleague joins our development team, her or his first task to do is to read through this cookbook. After reading and understanding each and every line, it should not be a problem to join any of our projects. Since Everit provides many OpenSource tools and components, the curriculum of Everit became OpenSource as well. To be able to work with our OSS solutions, it is important to study the content of this book first.

The documentation changes from time to time. When a technology becomes deprecated, the related sections will be grased from the documentation. As the book is stored in a Git repository, it is possible to find its older revisions as well. Therefore, when someone joins a project, for which older technologies are required, it is possible to find the right the revision of the curriculum. E.g.: It might happen that we change the build tool we use from Maven to Gradle. In that case, all Maven based chapters will be rewritten, but the earlier chapters about Maven will be available from the revision history.

Our architecture is currently based on OSGi. Therefore, it is very important to have a basic understanding of the way OSGi works.

How to follow?

To be able to follow the changes of the document, start watching the Github repository.

In case you are interested in the announcements of the newest releases, follow us on Twitter

How to contribute?

The documentation is available at Github. This means that the contribution should be done in the same way as it is done in the case of other Github based projects. You should fork, modify and send a pull request. If you find a spelling mistake, or you think you can provide additional information concerning any of the subjects included in the document, please do not hesitate to contribute!

To get a contribution accepted, please make sure that you keep the coding conventions introduced in the tutorials:

  • Lines should not be longer than 120 characters

  • Use two spaces for indentation

  • If it is possible, use Eclipse HTML editor with the formatting options listed above.
